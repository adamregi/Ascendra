# ADR 004: Decoupled Video Meeting Infrastructure using 100ms

## Status
Accepted

## Context
Ascendra's primary features include hosting video meetings and tracking attendee compliance (verifying that members join meetings and stay for minimum required durations). 

We need a secure, scalable video conferencing infrastructure. Handling video streams and managing call connections directly in our backend is impractical. We need to integrate a third-party video provider while ensuring that access is secure and attendee logs are accurate.

## Decision
We decided to integrate **100ms** as our video conference provider. We manage meeting lifecycles and track attendance through server-side token generation and webhooks.

1. **Server-Side Token Generation**: The Flutter client does not authenticate directly with 100ms. Instead, it calls the `/schedule-meeting` Edge Function, which retrieves entry tokens from the 100ms API using server-side credentials.
2. **Webhook Tracking**: We configure 100ms webhooks to notify our backend of room state changes (`room.started`, `room.ended`, `peer.joined`, `peer.left`).
3. **Webhook Handler**: Webhooks are routed to our `/record-attendance` endpoint, which validates payloads and updates `attendance_events` and `meeting_attendances` in the database.
4. **Decoupled Summarization**: When a `room.ended` webhook is received, the backend updates the meeting state and publishes a `MeetingEnded` event to process RAG summaries and audit attendance compliance.

## Consequences

### Positive Impacts
- **Scalable Video Routing**: Offloads video processing and network bandwidth to 100ms's global infrastructure.
- **Secure Access Control**: Meeting tokens are short-lived and generated on demand, preventing unauthorized room access.
- **Reliable Compliance Logs**: Relying on server-to-server webhook events ensures that join/leave logs are accurate, even if a user's app crashes or they disconnect unexpectedly.

### Trade-offs & Cons
- **Third-Party Dependency**: System availability depends on 100ms's uptime.
- **Webhook Delivery Latency**: Network delays in webhook delivery can cause small delays in updating real-time attendance states.
- **Recording Maintenance**: 100ms storage policies delete recordings after two days. The backend must download or migrate required recording files before they are removed.
